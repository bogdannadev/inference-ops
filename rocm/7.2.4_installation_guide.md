# Installing ROCm 7.2.4 on AMD Instinct MI350X DigitalOcean Droplet with Ubuntu 24.04

This guide documents a working installation path for **ROCm 7.2.4** on an **AMD Instinct MI350X DigitalOcean GPU Droplet** running **Ubuntu 24.04**.

It also includes the correct way to run **ROCm Bandwidth Test / TransferBench**. In ROCm 7.2.x, `rocm-bandwidth-test` is a plugin-based framework; the actual bandwidth benchmark is normally provided by the **TransferBench** plugin (`tb`).

---

## Why ROCm 7.2.4?

ROCm 7.2.4 is recommended for this setup because AMD lists MI350X compatibility with AMD GPU driver `30.30.x` up to `30.30.4`, while ROCm 7.2.3 lists support only up to `30.30.2` for that driver family.

MI350X is a CDNA4 / `gfx950` GPU. ROCm 7.2.4 supports this target in the AMD Instinct stack.

# Key Finding

For this DigitalOcean MI350X Ubuntu 24.04 droplet, installing ROCm with `--no-dkms` was **not sufficient**:

```bash
sudo amdgpu-install --usecase=rocm --no-dkms --no-32
```

That path left the system using the stock Ubuntu in-kernel `amdgpu` module:

```text
/lib/modules/6.8.0-124-generic/kernel/drivers/gpu/drm/amd/amdgpu/amdgpu.ko.zst
```

With the stock Ubuntu driver, the MI350X PCI device was visible, but the driver did **not** bind to the GPU:

```bash
readlink /sys/bus/pci/devices/0000:83:00.0/driver || echo "No driver bound"
```

Example failed output:

```text
No driver bound
```

The failing `dmesg` output looked like this:

```text
amdgpu 0000:83:00.0: amdgpu: Fatal error during GPU init
amdgpu: Doesn't get msg:1 from pf, error=-62
amdgpu: Doesn't get TRN_MSG_ACK from pf in 500 msec
amdgpu: probe of 0000:83:00.0 failed with error -22
```

The working fix was to install ROCm **with the AMD DKMS driver**:

```bash
sudo amdgpu-install --usecase=rocm --no-32
```

# Step 0 — Clean Previous Broken Install

If you previously used a runfile, mixed ROCm versions, or installed with `--no-dkms`, clean the system first.

```bash
sudo amdgpu-uninstall || true

sudo apt purge -y 'amdgpu*' 'rocm*' 'hip*' 'hsa*' 'rocr*' 'rocblas*' 'rocminfo*' 'rocm-smi*'

sudo apt autoremove -y
sudo apt autoclean

sudo rm -rf /opt/rocm*
sudo rm -rf /etc/rocm*
sudo rm -rf /usr/local/rocm*
sudo rm -rf /var/lib/dkms/amdgpu
sudo rm -rf /usr/src/amdgpu*
```

Reboot:

```bash
sudo reboot
```

### Why this matters

ROCm, AMDGPU DKMS, HIP, HSA, and ROCm user-space libraries must match. Mixing runfile packages, Ubuntu stock packages, and AMD repo packages can leave incompatible modules or libraries on the system.

# Step 1 — Verify System and GPU Visibility

After reboot:

```bash
lsb_release -a
uname -r
lspci | grep -i amd
lspci -k | grep -A 5 -i amd
```

Expected GPU visibility:

```text
83:00.0 Processing accelerators: Advanced Micro Devices, Inc. [AMD/ATI] Device 75b0
```

At this point, it is okay if the driver is not active yet.

---

# Step 2 — Install Base Dependencies

```bash
sudo apt update
sudo apt upgrade -y

sudo apt install -y \
    wget \
    curl \
    gnupg2 \
    gpg \
    ca-certificates \
    pciutils \
    kmod \
    dkms \
    build-essential \
    linux-headers-$(uname -r) \
    linux-modules-extra-$(uname -r) \
    linux-firmware \
    libnuma1 \
    numactl
```

### Why this matters

- `dkms` is required for building the AMDGPU DKMS driver.
- `linux-headers-$(uname -r)` must match the running kernel.
- `build-essential` provides the compiler toolchain.
- `linux-firmware` provides firmware blobs.
- `libnuma1` and `numactl` are required or useful for TransferBench-style bandwidth testing.

---

# Step 3 — Add the Official AMD ROCm 7.2.4 Repository

Download the official AMD `amdgpu-install` package for **ROCm 7.2.4** and **Ubuntu 24.04 Noble**:

```bash
cd /root

wget https://repo.radeon.com/amdgpu-install/7.2.4/ubuntu/noble/amdgpu-install_7.2.4.70204-1_all.deb
```

Install it:

```bash
sudo apt install -y ./amdgpu-install_7.2.4.70204-1_all.deb
sudo apt update
```

# Step 4 — Install ROCm 7.2.4 with AMD DKMS Driver

This is the critical step.

Use:

```bash
sudo amdgpu-install --usecase=rocm --no-32
```

Do **not** use `--no-dkms` for this tested MI350X DigitalOcean setup.

Correct:

```bash
sudo amdgpu-install --usecase=rocm --no-32
```

Incorrect for this tested setup:

```bash
sudo amdgpu-install --usecase=rocm --no-dkms --no-32
```

### Why this matters

Using `--no-dkms` left the droplet on Ubuntu's stock kernel `amdgpu` driver, which did not bind correctly to the MI350X VF. Installing with DKMS provided the correct AMDGPU driver stack.

---

# Step 5 — Check DKMS Status

Before rebooting, verify that DKMS built and installed the AMDGPU module:

```bash
dkms status
```

You should see an installed AMDGPU DKMS entry.

Also check where the `amdgpu` module exists:

```bash
find /lib/modules/$(uname -r) -name "amdgpu.ko*" | sort
```

A successful DKMS install should provide an AMD DKMS module, usually under an `updates/dkms`-style path.

---

# Step 6 — Rebuild initramfs

```bash
sudo update-initramfs -u -k all
```

### Why this matters

This ensures the new AMDGPU kernel module and firmware references are available during boot.

---

# Step 7 — Reboot

```bash
sudo reboot
```

Do not skip this step.

A working system should load the AMD DKMS driver, and `amd-smi` should later report something like:


# Step 11 — Verify PCI Driver Binding

```bash
readlink /sys/bus/pci/devices/0000:83:00.0/driver || echo "No driver bound"
```

Expected:

```text

```

```bash
lspci -k | grep -A 5 -i amd
```

Expected:


```text

```

```text

```

then the module exists but did not attach to the device.

```bash
rocm-smi
```

# Step 15 — Configure ROCm Bandwidth Test Environment

ROCm Bandwidth Test is plugin-based. Set the ROCm 7.2.4 environment explicitly:

```bash
export ROCM_PATH=/opt/rocm-7.2.4
export PATH=$ROCM_PATH/bin:$PATH
export LD_LIBRARY_PATH=$ROCM_PATH/lib:$ROCM_PATH/lib64:$ROCM_PATH/lib/rocm_bandwidth_test/plugins:${LD_LIBRARY_PATH}
```

---

# Step 16 — Install / Reinstall ROCm Bandwidth Test

```bash
sudo apt update
sudo apt install --reinstall -y rocm-bandwidth-test libnuma1 numactl
```

---

# Step 17 — Verify Bandwidth Test Plugins

Check plugin files:

```bash
find /opt/rocm* -type f \( -name "*transfer*" -o -name "*.amdplug" \) | sort
```

Expected relevant files include:

```text
/opt/rocm-7.2.4/lib/rocm_bandwidth_test/plugins/amd_hello.amdplug
/opt/rocm-7.2.4/lib/rocm_bandwidth_test/plugins/builtin.amdplug
/opt/rocm-7.2.4/lib/rocm_bandwidth_test/plugins/transferbench.amdplug
```

Check for missing shared libraries:

```bash
ldd /opt/rocm-7.2.4/lib/rocm_bandwidth_test/plugins/transferbench.amdplug | grep "not found" || echo "No missing shared libraries"
```

List registered plugins:

```bash
/opt/rocm-7.2.4/bin/rocm-bandwidth-test plugin -i
```

Expected: the TransferBench plugin should appear, typically with short name `tb`.

---

# Step 18 — Run ROCm Bandwidth Test

Once the TransferBench plugin is registered, run:

```bash
/opt/rocm-7.2.4/bin/rocm-bandwidth-test run tb a2a 1G | tee rocm-bandwidth-test-mi350x-a2a-1g.log
```

Meaning:

- `run tb` runs the TransferBench plugin.
- `a2a` is an all-to-all preset.
- `1G` uses a fixed 1 GiB transfer size.

For a transfer-size sweep:

```bash
/opt/rocm-7.2.4/bin/rocm-bandwidth-test run tb a2a 0 | tee rocm-bandwidth-test-mi350x-a2a-sweep.log
```

Use plugin help if needed:

```bash
/opt/rocm-7.2.4/bin/rocm-bandwidth-test run tb --help
```

# Troubleshooting

## Problem: `amd-smi` says no GPU devices

Example:

```text
Unable to detect any GPU devices
```

Check whether the PCI device is bound:

```bash
readlink /sys/bus/pci/devices/0000:83:00.0/driver || echo "No driver bound"
```

If output is:

```text
No driver bound
```

then AMDGPU did not bind to the GPU.

---

## Problem: `lspci` only shows `Kernel modules: amdgpu`

Bad:

```text
Kernel modules: amdgpu
```

Good:

```text
Kernel driver in use: amdgpu
Kernel modules: amdgpu
```

If only `Kernel modules` appears, the module exists but failed to attach.

---

## Problem: Stock Ubuntu amdgpu driver is loaded

Check:

```bash
modinfo amdgpu | grep filename
```

Bad example:

```text
/lib/modules/6.8.0-124-generic/kernel/drivers/gpu/drm/amd/amdgpu/amdgpu.ko.zst
```

Fix:

```bash
sudo amdgpu-install --usecase=rocm --no-32
sudo update-initramfs -u -k all
sudo reboot
```

---

## Problem: PF/TRN error in dmesg

Bad example:

```text
amdgpu: Doesn't get msg:1 from pf, error=-62
amdgpu: Doesn't get TRN_MSG_ACK from pf in 500 msec
amdgpu: probe of 0000:83:00.0 failed with error -22
```

In this tested setup, this happened when the stock Ubuntu `amdgpu` driver was used. Installing the AMD DKMS driver fixed it.

If this error persists even after AMD DKMS is loaded, contact DigitalOcean support and ask them to verify host-side MI350X PF/GIM/firmware compatibility.

---

## Problem: `rocm-bandwidth-test plugin -i` only shows `Hello`

If:

```bash
/opt/rocm-7.2.4/bin/rocm-bandwidth-test plugin -i
```

shows only:

```text
Hello
```

but this file exists:

```text
/opt/rocm-7.2.4/lib/rocm_bandwidth_test/plugins/transferbench.amdplug
```

then the plugin is present but not loading.

Set environment variables:

```bash
export ROCM_PATH=/opt/rocm-7.2.4
export PATH=$ROCM_PATH/bin:$PATH
export LD_LIBRARY_PATH=$ROCM_PATH/lib:$ROCM_PATH/lib64:$ROCM_PATH/lib/rocm_bandwidth_test/plugins:${LD_LIBRARY_PATH}
```

Check missing libraries:

```bash
ldd $ROCM_PATH/lib/rocm_bandwidth_test/plugins/transferbench.amdplug | grep "not found" || echo "No missing shared libraries"
```

Reinstall RBT and dependencies:

```bash
sudo apt update
sudo apt install --reinstall -y rocm-bandwidth-test libnuma1 numactl
```

Then check plugins again:

```bash
$ROCM_PATH/bin/rocm-bandwidth-test plugin -i
```

# Key Lesson

For **DigitalOcean MI350X on Ubuntu 24.04**, the working install command was:

```bash
sudo amdgpu-install --usecase=rocm --no-32
```

not:

```bash
sudo amdgpu-install --usecase=rocm --no-dkms --no-32
```

The `--no-dkms` path left the system using Ubuntu's stock `amdgpu` kernel driver, which did not bind correctly to the MI350X VF. Installing the AMD DKMS driver resolved the issue.
